# Request format for Sam vs Rizom

Use this to keep personal writing and Rizom writing separate.

## 1) State the owner
- **For Sam**
- **For Rizom**

## 2) State the task
Be explicit about the deliverable:
- draft
- rewrite
- edit
- summarize
- turn notes into copy
- create a post
- write a case study
- write a bio
- write homepage copy

## 3) State the audience
Who is this for?
- readers
- clients
- prospects
- collaborators
- internal team
- public website visitors

## 4) State the goal
What should the piece do?
- persuade
- explain
- announce
- position
- inform
- sharpen voice
- sell
- organize thinking

## 5) Provide source material
Attach or paste any of the following:
- reference docs
- notes
- drafts
- examples
- links
- screenshots

## 6) Optional voice guidance
Add any of these if useful:
- tone
- level of polish
- what to avoid
- examples to emulate
- examples to reject

## 7) If the task is for Rizom, say so clearly
Use phrases like:
- "for Rizom"
- "Rizom voice"
- "client-facing"
- "agency site copy"

## 8) If the task is for Sam, say so clearly
Use phrases like:
- "for Sam"
- "personal voice"
- "sharp and opinionated"
- "less corporate"

## 9) Example prompts
### For Sam
"For Sam: draft a short personal essay about AI and judgment. Audience: site readers. Goal: opinionated, sharp, and a little mischievous."

### For Rizom
"For Rizom: write homepage copy for the branding agency. Audience: potential clients. Goal: clear positioning, confident, premium, and not too playful."

### For Sam with source docs
"For Sam: rewrite this draft into a cleaner post. Keep the voice informal and sharp. Use the attached notes as source material."

### For Rizom with source docs
"For Rizom: turn these notes into a polished case study. Keep it client-facing and concise."

## 10) Shortcut rule
If you do not specify **Sam** or **Rizom**, I should ask before writing when the voice matters.
